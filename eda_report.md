# Data Science Bowl 2018 — EDA 보고서

---

## 1. 개요

본 보고서는 Data Science Bowl 2018 데이터셋에 대한 탐색적 데이터 분석(EDA) 결과를 정리한 것이다.  
이 대회의 목표는 다양한 조건(세포 종류, 배율, 이미징 방식)에서 촬영된 현미경 이미지에서  
**세포 핵(nucleus)을 instance segmentation**하는 모델을 개발하는 것이다.

| 항목             | 값                           |
| ---------------- | ---------------------------- |
| 학습 이미지 수   | 670                          |
| 테스트 이미지 수 | 65                           |
| 레이블 형식      | 핵 1개 = PNG 마스크 파일 1개 |
| 제출 형식        | Run-Length Encoding (RLE)    |

---

## 2. 데이터 구조

```
stage1_train/{ImageId}/
    images/{ImageId}.png   ← 원본 이미지 (1장)
    masks/{MaskId}.png     ← 핵 1개당 파일 1개 (수십~수백 개)

stage1_test/{ImageId}/
    images/{ImageId}.png   ← 원본 이미지만 존재 (masks 없음)
```

- 마스크 간 픽셀 중복 없음 (non-overlapping instance masks)
- 테스트셋은 이미지만 제공되며, 마스크를 예측해 RLE 형식으로 제출

---

## 3. 이미지 크기 분포

### 분석 내용

- 전체 670장의 너비(width) / 높이(height) 분포 확인
- (width, height) 고유 조합 수 집계
- 학습셋과 테스트셋의 크기 분포 비교

### 주요 관찰

- 이미지 크기가 **비정형(non-uniform)** 하다. 256×256처럼 작은 이미지부터 수백~1000px 이상의 이미지까지 혼재한다.
- 256×256이 가장 흔한 크기지만, 다양한 종횡비의 이미지가 존재해 **모델 입력 크기 표준화 또는 패딩 전략**이 필요하다.
- 테스트셋은 학습셋과 유사한 크기 범위를 갖지만, 학습에서 보지 못한 크기의 이미지가 포함될 수 있다.

### 모델링 시사점

> 고정 크기 입력을 요구하는 CNN 구조 사용 시, 이미지를 32의 배수로 패딩하거나 리사이즈하는 전처리가 필수적이다.

---

## 4. 채널 / 색상 모드

### 분석 내용

- PIL Image의 mode 속성으로 RGB / RGBA / L(그레이스케일) 구분
- R == G == B 조건으로 실질적 그레이스케일 이미지 판별

### 주요 관찰

- 대부분의 이미지는 **RGB 3채널**이지만, 그 중 상당수는 R=G=B인 **실질적 그레이스케일** 이미지다.
- 이미징 방식에 따라 다음과 같이 구분된다:

| 이미징 방식            | 특징                    | 채널                |
| ---------------------- | ----------------------- | ------------------- |
| 형광(Fluorescence)     | 핵이 밝고 배경이 어두움 | 대부분 그레이스케일 |
| 밝은 시야(Brightfield) | H&E 염색 등 컬러        | RGB 3채널           |

### 모델링 시사점

> 그레이스케일과 컬러 이미지를 함께 처리하려면 입력 정규화 전략을 분리하거나, 그레이스케일을 3채널로 복제하는 전처리가 필요하다.

---

## 5. 핵(Nucleus) 개수 분포

### 분석 내용

- 이미지당 마스크 파일 수 = 핵 개수
- 이미지 면적(픽셀 수)과 핵 개수의 상관관계

### 주요 관찰

- 이미지당 핵 개수는 **최소 수 개 ~ 수백 개**까지 큰 편차를 보인다.
- 이미지 면적이 클수록 핵이 많은 경향이 있지만, 세포 밀도에 따라 같은 크기에서도 편차가 크다.
- 핵이 매우 많은 이미지(고밀도)는 개별 경계 분리(instance separation)가 특히 어렵다.

### 모델링 시사점

> 핵 개수가 많은 고밀도 이미지는 경계(border) 예측을 별도 헤드로 학습하는 전략(border-body 분리)이 효과적이다 — topcoders 솔루션이 채택한 방식.

---

## 6. 핵 면적(Nucleus Area) 분포

### 분석 내용

- 개별 마스크 파일의 양성 픽셀 수(핵 크기) 집계 (80개 이미지 샘플)
- raw 분포 + log 변환 분포 비교

### 주요 관찰

- 핵 크기 분포는 **오른쪽으로 크게 치우친(right-skewed)** 분포를 보인다.
- 대부분의 핵은 수백~수천 px² 범위지만, 극단적으로 큰 핵도 존재한다.
- log 변환 시 분포가 상대적으로 정규분포에 가까워진다.

### 모델링 시사점

> 핵 크기 편차가 매우 크므로, 다양한 스케일의 객체를 탐지하는 **FPN(Feature Pyramid Network)** 또는 **멀티스케일 앙상블**이 유리하다.

---

## 7. 마스크 커버리지 (Coverage)

### 분석 내용

- 이미지 전체 픽셀 중 핵이 차지하는 비율(%) 계산

### 주요 관찰

- 커버리지는 이미지마다 편차가 매우 크다.
- 형광 이미지는 배경이 대부분 검은색이라 커버리지가 낮은 경향이 있다.
- 고밀도 세포 이미지는 커버리지가 50% 이상에 달하는 경우도 있다.

### 모델링 시사점

> 배경 클래스 불균형이 이미지마다 다르므로, **Dice Loss** 또는 **Focal Loss**처럼 클래스 불균형에 강한 손실 함수가 필요하다.

---

## 8. 픽셀 강도(Intensity) 분포

### 분석 내용

- 100개 이미지 샘플의 R/G/B 채널별 평균 및 표준편차
- 8개 이미지의 픽셀 강도 히스토그램

### 주요 관찰

- 형광 이미지는 대부분의 픽셀이 0(검정)에 몰리고 핵 부분만 밝은 **이중 피크(bimodal)** 분포를 보인다.
- 밝은 시야(brightfield) 이미지는 배경이 밝고 핵이 어두운 **역전된 분포**를 보이는 경우가 많다.
- 채널 간 평균값 편차가 큰 이미지(컬러)와 R≈G≈B인 이미지(그레이스케일)가 혼재한다.

### 모델링 시사점

> 이미지별로 정규화 전략이 달라야 한다. **per-image normalization**(각 이미지를 독립적으로 0~1 정규화)이 global normalization보다 안정적이다.

---

## 9. 학습셋 vs 테스트셋 비교

### 주요 관찰

- 테스트셋(65장)은 학습셋(670장) 대비 크기가 작다.
- 이미지 크기 분포는 유사하지만, stage 2에서는 학습에서 보지 못한 실험 조건의 이미지가 추가될 예정이다.
- Public(대회 중 공개) / Private(대회 종료 후 공개) 분리로 Public 과적합을 방지한다.

---

## 10. 종합 요약 및 모델링 전략

### 데이터 특성 요약

| 특성           | 설명                            |
| -------------- | ------------------------------- |
| 이미지 크기    | 비정형, 256px ~ 1000px 이상     |
| 색상           | RGB 컬러 + 그레이스케일 혼재    |
| 핵 개수/이미지 | 수 개 ~ 수백 개, 편차 매우 큼   |
| 핵 크기        | 오른쪽 치우침, 로그 스케일 분포 |
| 커버리지       | 이미지마다 크게 다름            |
| 이미징 방식    | 형광 / 밝은 시야 혼재           |

### 권장 전처리 전략

1. **이미지 리사이즈 / 패딩** — 32의 배수로 맞춤
2. **per-image 정규화** — global 통계 사용 지양
3. **그레이스케일 → 3채널 복제** — 입력 채널 통일
4. **데이터 증강** — 좌우/상하 반전, 회전, 색상 지터

### 권장 모델 전략

1. **Encoder-Decoder 구조** (U-Net 계열) — 다양한 이미지 크기 처리
2. **FPN / 멀티스케일 특징** — 핵 크기 편차 대응
3. **Border + Body 분리 헤드** — 고밀도 이미지의 인접 핵 분리
4. **Dice Loss + Cross-Entropy 혼합** — 클래스 불균형 대응
5. **Test-Time Augmentation(TTA)** — 예측 안정성 향상
